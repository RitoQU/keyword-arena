## 一、问题诊断

### 1.1 当前架构的外部依赖

| 依赖 | 域名 | 连接方 | 中国大陆可达性 |
|------|------|--------|--------------|
| **Vercel 托管** | `*.vercel.app` | 用户浏览器 | ❌ **完全不可达** — DNS 污染 + IP 封锁 |
| **Google Fonts** | `fonts.googleapis.com` / `fonts.gstatic.com` | 用户浏览器 | ❌ **完全不可达** — GFW 长期封锁 |
| **Supabase** | `ugyznwvkmyqbkvvtqipi.supabase.co` | 服务端 API Route | ⚠️ 服务端可达，但延迟较高（~200ms） |
| **MiniMax API** | `api.minimax.chat` | 服务端 API Route | ✅ **国内公司**，完全可达 |

### 1.2 核心结论

中国用户无法访问的根本原因有两个：
1. **Vercel 被 GFW 封锁** — 页面根本加载不出来
2. **Google Fonts 被 GFW 封锁** — 即使页面加载了，像素风字体也无法显示

好消息：
- 所有 API 调用（Supabase、MiniMax）都在**服务端**完成，浏览器不直接连接这些服务
- 浏览器除了 Google Fonts 外，**零外部请求**（音频、图片全部本地托管）
- 这意味着只要解决 **托管平台** 和 **字体** 两个问题，游戏即可在中国正常运行

## 二、方案对比

### 方案 A：香港 VPS 部署（⭐ 推荐 — 最佳性价比）

将应用部署到香港地区的云服务器，无需 ICP 备案。

| 维度 | 评估 |
|------|------|
| 实施周期 | **1-2 天** |
| 月费 | **¥30-70/月**（轻量服务器） |
| ICP 备案 | **不需要**（香港不受大陆备案制度约束） |
| 延迟 | 华南 ~20-40ms，华北 ~50-80ms，**完全可用** |
| 可用性 | 主流云商的香港线路稳定，被封概率极低 |
| MiniMax 延迟 | ~15-30ms（优于当前 Vercel 部署的 ~200ms） |
| Supabase 延迟 | ~100-200ms（取决于 Supabase 项目所在区域） |
| 维护成本 | 需自行管理服务器（Docker 部署可最小化运维） |

**供应商选择：**

| 供应商 | 产品 | 配置 | 月费 | 推荐度 |
|--------|------|------|------|--------|
| 阿里云 | 轻量应用服务器（香港） | 2C2G | ~¥24/月 | ⭐⭐⭐ 最稳定 |
| 腾讯云 | 轻量应用服务器（香港） | 2C2G | ~¥32/月 | ⭐⭐⭐ |
| Bandwagon（搬瓦工） | THE PLAN（HK） | 2C1G | ~$50/年 ≈ ¥30/月 | ⭐⭐ 性价比高 |

### 方案 B：国内 VPS + ICP 备案（最佳体验，周期长）

部署到大陆服务器，配合 ICP 备案和自有域名。

| 维度 | 评估 |
|------|------|
| 实施周期 | **3-5 周**（备案审批 2-4 周） |
| 月费 | **¥30-100/月**（服务器）+ **¥50/年**（域名） |
| ICP 备案 | **必须**（大陆服务器 80/443 端口不备案不开放） |
| 延迟 | 全国 ~10-30ms，**最优** |
| 合规性 | **完全合规**，长期运营无忧 |
| 限制 | 需个人身份证 + 实名域名 + 等待审批 |

> ⚠️ 备案期间网站无法上线，不适合需要快速上线的场景。

---

### 方案 C：Cloudflare Pages（免费，效果不确定）

迁移到 Cloudflare Pages，利用其全球 anycast 网络。

| 维度 | 评估 |
|------|------|
| 实施周期 | **半天** |
| 月费 | **免费** |
| ICP 备案 | 不需要 |
| 延迟 | 不稳定，部分地区/运营商可达，部分不可达 |
| 风险 | `.pages.dev` 域名也存在被封的风险 |

> ⚠️ Cloudflare 免费版不含中国大陆加速节点（需企业版 + JD Cloud 合作），实际从中国访问路由到东京/新加坡节点，速度勉强可用但不稳定。**不推荐作为唯一方案。**

---

### 方案 D：双部署 + 智能 DNS（终极方案）

保留 Vercel 服务国际用户，增加国内/香港节点服务中国用户。

| 维度 | 评估 |
|------|------|
| 实施周期 | 方案 A 基础上 + 1 天 |
| 月费 | Vercel（免费）+ VPS（¥30-70） + 域名 DNS（¥50/年）|
| 效果 | 国内外用户都获得最优体验 |
| 复杂度 | 需要维护双份部署，需自有域名 |

## 四、Phase 1 详细执行计划

### 4.1 自托管 Google Fonts（代码改动）

当前 `layout.tsx` 通过 `<link>` 加载两个 Google Fonts：
- **Press Start 2P**（英文像素字体）
- **ZCOOL QingKe HuangYou**（中文像素字体）

改造方式：
1. 下载字体文件 `.woff2` 到 `public/fonts/` 目录
2. 在 CSS 中用 `@font-face` 本地引用
3. 移除 `layout.tsx` 中的 Google Fonts `<link>` 标签

**字体文件大小估算：**
| 字体 | 大小 | 说明 |
|------|------|------|
| Press Start 2P | ~7KB | 仅 Latin 字符集 |
| ZCOOL QingKe HuangYou | ~4MB | 中文字符集较大 |

> 总计约 4MB，首次加载会增加一些时间，但一旦缓存后无影响。可通过 `font-display: swap` 优化首屏感知速度。

### 4.2 香港 VPS 部署

**推荐配置：**
- 阿里云轻量应用服务器-香港
- 2C2G，40GB SSD
- ~¥24/月

**部署架构：**
```
用户浏览器 (中国大陆)
    │
    │  HTTPS (IP 直连 or 自有域名)
    ▼
┌──────────────────────────┐
│   Hong Kong VPS          │
│   ┌──────────────────┐   │
│   │  Docker           │   │
│   │  ├─ Node.js 20    │   │
│   │  ├─ Next.js App   │   │
│   │  └─ PM2 进程管理   │   │
│   └──────────────────┘   │
│   ┌──────────────────┐   │
│   │  Nginx            │   │
│   │  ├─ SSL 终端       │   │
│   │  ├─ Gzip 压缩     │   │
│   │  └─ 静态文件缓存   │   │
│   └──────────────────┘   │
└──────────┬───────────────┘
           │
     ┌─────┴──────┐
     ▼            ▼
  Supabase    MiniMax API
 (数据库)     (角色生成)
```

**部署步骤：**

```bash
# 1. 在 VPS 上安装 Docker
curl -fsSL https://get.docker.com | sh

# 2. 克隆项目
git clone <repo-url> /opt/keyword-arena
cd /opt/keyword-arena/keyword-arena-app

# 3. 创建 .env.local（从本地安全传输）
scp .env.local root@<vps-ip>:/opt/keyword-arena/keyword-arena-app/

# 4. 构建并启动
docker build -t keyword-arena .
docker run -d --name keyword-arena -p 3000:3000 --env-file .env.local keyword-arena

# 5. Nginx 反向代理 + Let's Encrypt SSL（如有域名）
```

**需要创建的文件：**
- `Dockerfile`
- `docker-compose.yml`
- `.dockerignore`
- `nginx.conf`（如使用 Nginx）

### 4.3 HTTPS 方案

| 场景 | 方案 | 说明 |
|------|------|------|
| 有自有域名 | Let's Encrypt + Nginx | 免费 SSL，自动续期 |
| 无域名，IP 直连 | 自签名证书 | 浏览器会提示不安全，但可用 |
| 临时测试 | HTTP 直连 | 仅限测试，不推荐生产 |

> 建议至少注册一个 `.com` 域名（~¥50/年），便于 HTTPS 和后续 DNS 分流。

## 六、对现有功能的影响评估

| 功能 | 影响 | 说明 |
|------|------|------|
| 角色生成 | ✅ 无影响 | MiniMax API 从香港访问更快 |
| 对战系统 | ✅ 无影响 | 纯服务端逻辑 |
| 登录/账号 | ✅ 无影响 | Supabase 服务端访问 |
| 排行榜/历史 | ✅ 无影响 | Supabase 服务端访问 |
| 音效/音乐 | ✅ 无影响 | 本地文件，已在 public/ 目录 |
| 像素风字体 | ✅ 改善 | 自托管后不再依赖 Google |
| Admin 面板 | ✅ 无影响 | 服务端 API |
| **数据库** | ⚠️ 共享 | HK 和 Vercel 共用同一个 Supabase，数据完全一致 |

> **关键：** 由于 Supabase 数据库是共享的，无论用户从 Vercel 还是 HK VPS 访问，看到的是同一份数据（排行榜、对战记录等）。不需要数据同步。

## 七、风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|------|----------|
| 香港 VPS IP 被 GFW 封锁 | 低（主流云商风险极低） | 选阿里云/腾讯云等大厂，被封可换 IP |
| Supabase 从 HK 访问变慢 | 中 | 可将 Supabase 项目迁移到亚洲区域 |
| 需要维护两套部署 | 确定 | Docker 标准化 + 简单的部署脚本即可 |
| VPS 被攻击 | 低 | 启用防火墙，仅开放 80/443 端口 |
| 中文字体文件较大 (4MB) | 确定 | `font-display: swap` + Nginx 长缓存 + gzip 压缩约 2MB |
